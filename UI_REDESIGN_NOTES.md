# UI Redesign Notes｜漫畫閱讀體驗改版重點

本次改版將網站從「首頁展示型 20/80 版面」調整為「首頁展示 + 漫畫閱讀優先」的雙模式架構。

## 核心調整

1. **閱讀模式自動收合頂部區塊**  
   進入角色設定或任一主題漫畫時，頂部標題、版權、語言、版型、創作動機與藝廊入口會自動收合為薄型控制列，釋放垂直空間。

2. **漫畫改為適合寬度顯示**  
   舊版以 `object-fit: contain` 受限於閱讀器高度，直式漫畫容易顯示過小。新版在閱讀模式中改為 `width: 100%`，圖片可填滿閱讀區寬度，直式頁面以垂直捲動閱讀。

3. **CSS 沉浸閱讀取代強制 Fullscreen API**  
   舊版全螢幕在部分瀏覽器會限制網頁縮放或讓圖片仍受高度限制。新版「沉浸閱讀」使用 CSS fixed overlay，不強制進入瀏覽器原生全螢幕，保留瀏覽器縮放、觸控與鍵盤操作彈性。

4. **內建閱讀縮放**  
   新增縮小、100%、放大、適合寬度，並支援 `+`、`-`、`0` 快捷鍵與雙擊放大。

5. **控制列固定底部**  
   漫畫閱讀時，上一張、下一張、縮放與沉浸閱讀控制集中於底部工具列，視線可以停留在漫畫頁面。

6. **效能邊界**  
   首頁不預載十個主題全部圖片；進入主題後只渲染單一 `<img>`，並只預載前後相鄰頁，維持 GitHub Pages 純靜態與低資源負擔。

## 建議後續優化

- 將每張漫畫建立 1024w、1536w、2048w 三種尺寸，搭配 `srcset` 與 `sizes`，手機可載入較小圖，桌機可載入高清圖。
- 若未來每個主題超過 5 頁，可加入縱向連續閱讀模式與閱讀進度記憶。
- 若漫畫內文字較小，可另外產出 2x 高解析版本供桌機與大螢幕使用。
